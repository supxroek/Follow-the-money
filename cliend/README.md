# LINE Shared Expense Tracker - Client\n\nReact-based frontend client for the LINE Shared Expense Tracker application using LINE LIFF (LINE Front-end Framework).\n\n## Features\n\n- **LINE LIFF Integration**: Seamless authentication and messaging through LINE\n- **Expense Management**: Create, edit, and track shared expenses\n- **Group Management**: Create and manage expense groups\n- **Debt Tracking**: Monitor who owes what and facilitate settlements\n- **Real-time Updates**: Get notifications for expense updates\n- **Mobile-First Design**: Optimized for mobile devices\n\n## Tech Stack\n\n- **React 19** - UI framework\n- **Vite** - Build tool and development server\n- **TailwindCSS** - Styling and responsive design\n- **React Router** - Client-side routing\n- **Zustand** - State management\n- **React Hook Form** - Form handling\n- **Axios** - HTTP client\n- **LINE LIFF SDK** - LINE platform integration\n\n## Development Setup\n\n### Prerequisites\n\n- Node.js 18+ and npm\n- LINE Developers account\n- LIFF app configured in LINE Developers Console\n\n### Installation\n\n1. **Clone and install dependencies:**\n   ```bash\n   cd cliend\n   npm install\n   ```\n\n2. **Environment Configuration:**\n   ```bash\n   cp .env.example .env\n   ```\n   \n   Update `.env` with your configuration:\n   ```env\n   VITE_LIFF_ID=your_liff_id_here\n   VITE_API_BASE_URL=http://localhost:3001/api\n   VITE_NODE_ENV=development\n   VITE_LOG_LEVEL=DEBUG\n   ```\n\n3. **Start development server:**\n   ```bash\n   npm run dev\n   ```\n\n4. **For LIFF development (HTTPS required):**\n   ```bash\n   npm run serve:https\n   ```\n\n### Available Scripts\n\n- `npm run dev` - Start development server\n- `npm run build` - Build for production\n- `npm run preview` - Preview production build\n- `npm run lint` - Run ESLint\n- `npm run lint:fix` - Fix ESLint issues\n- `npm run serve:https` - Start HTTPS development server for LIFF\n\n## Project Structure\n\n```\nsrc/\n├── components/          # Reusable UI components\n│   ├── Button.jsx\n│   ├── Input.jsx\n│   ├── Modal.jsx\n│   ├── Loading.jsx\n│   ├── Toast.jsx\n│   └── ErrorBoundary.jsx\n├── layouts/             # Layout components\n│   └── MainLayout.jsx\n├── pages/               # Page components\n│   ├── Dashboard.jsx\n│   ├── Login.jsx\n│   ├── Groups/\n│   ├── Expenses/\n│   └── Profile/\n├── stores/              # Zustand stores\n│   ├── authStore.js\n│   ├── groupStore.js\n│   ├── expenseStore.js\n│   └── debtStore.js\n├── services/            # API and external services\n│   ├── apiService.js\n│   └── liffService.js\n├── hooks/               # Custom React hooks\n│   └── useToast.js\n├── utils/               # Utility functions\n│   └── logger.js\n├── App.jsx              # Main app component\n└── main.jsx             # App entry point\n```\n\n## Key Features Implementation\n\n### Authentication\n- LINE LIFF authentication flow\n- Automatic token management\n- Protected routes\n\n### State Management\n- Zustand for global state\n- Separate stores for different domains\n- Persistent auth state\n\n### API Integration\n- Axios with automatic token refresh\n- Comprehensive error handling\n- Request/response interceptors\n\n### UI/UX\n- Mobile-first responsive design\n- Loading states and error boundaries\n- Toast notifications\n- Accessible components\n\n## LINE LIFF Configuration\n\n1. **Create LIFF App** in LINE Developers Console\n2. **Configure LIFF Settings:**\n   - Endpoint URL: Your app URL\n   - Size: Full or Compact\n   - Scope: profile, openid\n3. **Update environment variables** with LIFF ID\n\n## Building for Production\n\n```bash\nnpm run build\n```\n\nThe build artifacts will be stored in the `dist/` directory.\n\n## Deployment\n\n### Requirements\n- HTTPS (required for LIFF)\n- Static file hosting\n- Environment variable configuration\n\n### Recommended Platforms\n- Vercel\n- Netlify\n- AWS S3 + CloudFront\n- Firebase Hosting\n\n## Environment Variables\n\n| Variable | Description | Required |\n|----------|-------------|----------|\n| `VITE_LIFF_ID` | LINE LIFF App ID | Yes |\n| `VITE_API_BASE_URL` | Backend API URL | Yes |\n| `VITE_NODE_ENV` | Environment (development/production) | No |\n| `VITE_LOG_LEVEL` | Logging level (DEBUG/INFO/WARN/ERROR) | No |\n\n## Contributing\n\n1.
